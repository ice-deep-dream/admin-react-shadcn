# 项目规范

## Git 提交规范

### 提交信息格式

```
<type>(<scope>): <subject>
```

#### 提交类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `init` | 项目初始化 | `init: setup project structure` |
| `feat` | 新功能 | `feat(auth): add email login` |
| `fix` | Bug 修复 | `fix(api): handle 401 token expiration` |
| `perf` | 性能优化 | `perf: reduce bundle size by code splitting` |
| `refactor` | 代码重构 | `refactor: extract user service logic` |
| `style` | 代码格式调整 | `style: format components with Prettier` |
| `test` | 测试相关 | `test: add unit tests for cart module` |
| `docs` | 文档更新 | `docs: update API usage guide` |
| `chore` | 构建/工具/脚本 | `chore: update dev dependencies` |
| `deps` | 依赖版本变更 | `deps: upgrade React to 18.3.0` |
| `ci` | CI 配置 | `ci: add lint check in PR workflow` |
| `build` | 构建配置 | `build: set publicPath for CDN` |
| `workflow` | 开发流程/模板/规范 | `workflow: add issue template` |
| `revert` | 回滚提交 | `revert: feat(login)` |
| `release` | 发布版本 | `release: v1.4.0` |

### 分支命名规范

| 分支类型 | 命名规范 | 用途 |
|--------|--------|------|
| `master` | `master` | 生产环境代码 |
| `develop` | `develop` | 开发主干 |
| `feature` | `feature/<name>` | 新功能开发 |
| `bugfix` | `bugfix/<desc>` | Bug 修复 |
| `hotfix` | `hotfix/<desc>` | 紧急线上修复 |
| `release` | `release/vX.Y.Z` | 发布准备 |

## 代码规范

### TypeScript 规范

- 使用 TypeScript 进行类型检查
- 函数和组件必须定义类型
- 避免使用 `any` 类型
- 使用接口定义复杂数据结构

### 组件命名规范

- 组件文件名使用大驼峰命名法（PascalCase）
- 组件名与文件名保持一致
- 使用函数式组件

### 样式规范

- 使用 Tailwind CSS 进行样式编写
- 保持样式的一致性和可复用性
- 避免过度嵌套的样式

## 国际化规范

### 翻译文件结构

```
src/locales/
├── zh.json  # 中文翻译
└── en.json  # 英文翻译
```

### 翻译键命名规范

- 使用小写字母和点号分隔
- 采用模块化组织，如 `common.xxx`、`auth.xxx`
- 保持翻译键的语义清晰

### 默认语言

- 默认语言：中文（zh）
- 国际化库：i18next + react-i18next

## 构建规范

### 开发环境

```bash
pnpm install
pnpm run dev
```

### 生产构建

```bash
pnpm run build
pnpm run preview
```

## 测试规范

- 单元测试：使用 Vitest
- 端到端测试：使用 Playwright
- 测试覆盖率：保持在 80% 以上

## 文档规范

- 重要功能需要编写文档
- 保持文档的及时更新
- 使用 Markdown 格式编写文档
